## Config File

Polaris looks for configuration in the project root, in this order:

1. `polaris.config.ts`
2. `polaris.config.js`
3. `polaris.config.json`

## Schema

```ts
export default {
  // Page title (shown in browser tab)
  title: "My Dashboard",

  // Project description
  description: "Business metrics dashboard",

  // Theme (currently only dark is supported)
  theme: "dark",

  // Base path for deployment (e.g., "/repo-name/" for GitHub Pages)
  basePath: "/",

  // Data directory settings
  data: {
    dir: "data",   // Relative to project root
  },

  // Build output settings
  build: {
    outDir: "dist", // Output directory for production build
  },

  // Semantic model directory
  models: {
    dir: "models", // Relative to project root
  },

  // AI settings
  ai: {
    provider: "anthropic",
    apiKey: undefined, // Or set POLARIS_AI_API_KEY env var
    model: "claude-sonnet-4-20250514",
  },
};
```

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `title` | `string` | — | Project title |
| `description` | `string` | — | Project description |
| `theme` | `"dark" \| "light"` | `"dark"` | Color theme |
| `basePath` | `string` | `"/"` | URL base path (for subpath deploys) |
| `data.dir` | `string` | `"data"` | Data files directory |
| `build.outDir` | `string` | `"dist"` | Build output directory |
| `models.dir` | `string` | `"models"` | Directory for semantic model YAML files |
| `ai.provider` | `string` | `"anthropic"` | AI provider |
| `ai.apiKey` | `string` | — | API key (optional; can also set `POLARIS_AI_API_KEY` env var) |
| `ai.model` | `string` | `"claude-sonnet-4-20250514"` | Model to use for AI features |
