---
title: Slider
description: A range slider for selecting a numeric value within a bounded range.
---

## Props

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| name | string | Yes | — | Filter variable name. Use `${name}` in SQL to reference the numeric value. |
| min | number | Yes | — | Minimum allowed value. |
| max | number | Yes | — | Maximum allowed value. |
| step | number | No | `1` | Step increment between values. |
| label | string | No | — | Label displayed above the slider. |

The current value is displayed to the right of the slider track. The initial value defaults to `min`.

## Example

````mdx
<Slider name="min_revenue" min={0} max={1000000} step={10000} label="Minimum Revenue" />

```sql
-- name: high_value_customers
SELECT
  customer_name,
  SUM(order_total) AS lifetime_revenue
FROM orders
GROUP BY customer_name
HAVING SUM(order_total) >= ${min_revenue}
ORDER BY lifetime_revenue DESC
```

<DataTable data="high_value_customers" />
````

### Combined with other inputs

````mdx
<Grid cols={3}>
  <Slider name="min_price" min={0} max={500} step={5} label="Min Price ($)" />
  <Slider name="max_price" min={0} max={500} step={5} label="Max Price ($)" />
  <Dropdown name="category" options={["Electronics", "Clothing", "Home"]} label="Category" />
</Grid>

```sql
-- name: filtered_products
SELECT product_name, category, price
FROM products
WHERE price >= ${min_price}
  AND price <= ${max_price}
  AND category = '${category}'
ORDER BY price
```

<DataTable data="filtered_products" />
````
